# Standardize home-screen side widget lists

## Goal

Move the existing Daily Gifts launcher out of the top-left home utility row and into a reusable left-side home widget list. Standardize both side rails as explicit widget lists so future home-screen buttons can be added to either side without introducing one-off positioning rules.

## Confirmed product decisions

- The home utility row must retain only the existing coin balance and profile controls. Remove **only** the Daily Gifts launcher from that row; do not otherwise change those controls or their placement.
- Create a **left widget list** that initially contains Daily Gifts only.
- Create a **right widget list** that contains the existing Leaderboard and Battle Pass entries, preserving their current top-to-bottom order: Leaderboard, then Battle Pass.
- Each side list is vertically centered as a whole within the home screen. Its buttons are vertically stacked using the existing right-widget button size and gap. Therefore, the single Daily Gifts button sits at the vertical center on the left.
- Preserve Daily Gifts’ established UI and behavior: its icon, accessible name (`Daily gifts`), dialog flow, visual states, keyboard operation, and focus restoration must remain unchanged. This is a structural/layout refactor, not a Daily Gifts redesign.
- Do not add product controls or alter unrelated home-screen navigation.

## Relevant implementation facts

- Home markup is in `public/index.html` under `#home-screen`.
  - `#daily-gifts-launcher.daily-gifts-launcher` currently appears in `.home-utility-row` before `#coin-holder` and `#open-profile`.
  - `#open-leaderboard.leaderboard-entry` and `#open-battle-pass.battle-pass-entry` are currently separate home-screen sibling buttons.
- Layout styling is in `public/css/styles.css`.
  - `#home-screen` is the positioned fixed design canvas and uses the project’s scaled/responsive home layout.
  - `.daily-gifts-launcher` currently has its own absolute positioning rule; the Leaderboard and Battle Pass entries also have existing side-entry styling/positioning. Replace this duplicated placement approach with shared list/container styling while retaining the established widget visuals.
- `public/js/view.js` resolves and binds the existing launcher/entry IDs and controls their visibility and focus on home navigation. In particular, `showHome()` unhides `dailyGiftLauncher`, and non-home transitions hide it; returning from the relevant screens can focus the leaderboard or battle-pass entry. Keep those IDs and behaviors compatible.
- Existing Daily Gifts coverage is in `tests/daily-gifts.test.js`; home layout/style checks are in `tests/styles.test.js` and related UI tests.
- Follow `AGENTS.md`: prefer Paper.css defaults; any custom CSS must be necessary, scoped to the game component, and must not override Paper.css globally.

## Scope of work

1. Refactor the `#home-screen` markup to introduce semantically/naming-consistent left and right widget-list containers.
   - Put `#daily-gifts-launcher` in the left list.
   - Put `#open-leaderboard` followed by `#open-battle-pass` in the right list.
   - Keep the coin holder and profile button in `.home-utility-row`.
   - Preserve all existing button IDs, button types, SVG/icon markup, ARIA attributes, and event-binding hooks unless a change is strictly needed to support the shared list structure.

2. Refactor home-side-widget CSS around shared list and item rules.
   - Position the left and right lists at their respective home-screen edges and vertically center each list as a group.
   - Stack list items with the same dimensions and inter-item spacing currently used by the right-side widgets.
   - Make left/right placement use the shared list abstraction rather than per-button absolute coordinates, so adding another widget is a straightforward list-item addition.
   - Preserve the app’s existing fixed home-canvas scaling and responsive behavior. Side widgets must remain usable/visible at supported viewport sizes and must not obstruct the core title, preview, Start game control, coin display, or profile control.
   - Preserve the current visual treatment of Daily Gifts, Leaderboard, and Battle Pass other than their agreed placement/container refactor.

3. Preserve interaction and navigation behavior.
   - Daily Gifts must still open the existing dialog and retain its current visibility, focus, keyboard, and dismissal behavior.
   - Leaderboard and Battle Pass must continue opening their existing screens/dialog flows; their return-focus behavior must still target their respective entry buttons.
   - Side widgets must only be shown while the home screen is active, consistent with the existing view lifecycle.

4. Add or update focused automated coverage for the standardized structure and layout contract, without relying solely on fragile incidental styling implementation details.

## Non-goals

- No redesign of widget icons, labels, colors, animations, dialogs, or their business logic.
- No changes to Daily Gifts claiming/reward rules or persistence.
- No changes to coin balance, profile functionality, gameplay, leaderboard behavior, or battle-pass behavior beyond preserving the affected entry controls.
- Do not introduce additional side widgets in this task.

## Acceptance criteria

- On the home screen, Daily Gifts appears on the left at the same vertical-center rail position that mirrors the right-side widget rail; it is no longer in the top utility row.
- The top utility row contains the existing coin balance and profile controls, but not the Daily Gifts launcher.
- The right list contains Leaderboard above Battle Pass, and the left list contains Daily Gifts.
- Both side rails are explicit reusable lists/containers with shared layout/item rules; adding a future button to either list does not require a bespoke absolute-positioning rule for that button.
- With one item in the left rail, Daily Gifts is vertically centered. With the two current right items, their combined stack is vertically centered and retains the established widget dimensions and gap.
- Daily Gifts has the same icon and accessible name and continues to open/close its existing dialog correctly, including keyboard/focus behavior.
- Leaderboard and Battle Pass remain operable, retain their ordering, and preserve navigation return focus.
- Widgets remain hidden when leaving the home screen and return when home is shown, matching current view lifecycle behavior.
- Existing home canvas scaling/responsiveness and unrelated top/home controls remain intact.
- Relevant automated tests pass, including the updated/additional structural/layout and existing Daily Gifts interaction coverage.
